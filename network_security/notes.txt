HTTPS:
    HTTPS = HTTP + SSL/TLS
        传输加密, 提供认证等机制
    端口与 HTTP 不同, 是 443

    无法避免 ARP攻击 和 DNS攻击, 但是内容无法被截取
    有效防止内容篡改, 篡改后解密是无效数据
    可以验证服务器身份

    会话劫持
        如果只在登陆时使用HTTPS, 之后不使用
        可以在登录后, 窃取 cookie, 直接利用仿造 cookie 假冒登陆

电子商务安全
    核心是网络支付
    要点
        - 支付账号和密码被窃取
        - 支付金额被更改
        - 支付双方真实性不确定
        - 抵赖, 修改...
        - 针对支付系统的攻击
        - 保密性
    SET 协议
        网上信用卡协议, x.509v3证书
        - 真实性
        - 保密性
        - 隐私性
        - 实时性
        参与方
        - 发卡行
        - 持卡人
        - 商家
        - 收款行
            商家和收款行互相信任
        - 证书权威
        - 支付网关
            支付网络和 SET 唯一的接口
    交易过程
        身份认证
        购买请求 顾客 - 商家
        支付授权 商家 - 支付网关 - 发卡行
        支付获取 商家 - 支付网关 - 发卡行

        双签名
            连接两个给不同接收者的报文
        PI  payment info
            应当对商家保密
        OI  order info
            应当对银行保密

入侵技术
    入侵者
        - 假冒者
        - 非法访问的
        - 秘密用户

    基于统计的入侵检测
        通过一段时间内的用户行为来建立统计模式
        分类
            - 阈值
            - 轮廓
    给予规则的入侵检测
        规则是通过专家系统定义的
    蜜罐技术
        诱导攻击者远离重要系统的圈套
        任何对蜜罐的攻击都会出现攻击成功的假象, 让攻击者继续攻击
        任何对蜜罐的访问都是可以的

恶意软件
    分类
        - 依赖宿主程序
            后门
                非正常的访问渠道
                常用于调试, 但被恶意利用
                常有权限提升, 远程执行命令
            逻辑炸弹
                一般是预先规定了发作时间
                这种规定时间, 规定动作, 触发攻击的属性, 成为逻辑炸弹属性
            木马
                表面看起来是很正常的 [很有用的] 程序,
                但是内部含有隐蔽代码
                如果只提供远程访问 -> 后门
                如果还伪装成良性程序 -> 木马
                欺骗常用手电
                    - 名称欺骗
                    - 文件捆绑 
                    - ...
            病毒
                一段可以通过修改自身, 修改其他程序的 [程序片段]
                有传染性
                多数都是基于特定方式工作的, 如某个 OS 或某个 硬件平台
                e.g. 引导性病毒
                    修改MBR
        - 独立宿主程序
            蠕虫
                可以自我复制的 [程序]
                通过网络上计算机系统漏洞, 完成自身的复制到其他计算机上
                - 无需宿主
                - 不伪装
                - 占用系统 / 资源, 破坏其他程序
                - 是 [网络] 上特有的
                常通过邮件, 远程执行/登陆来复制
            僵尸
                接管互联网上其他计算机, 通过他们发送攻击
                通常用在拒绝服务攻击上
    大致考察
        描述某一类恶意软件, 发作它的过程
        在这一过程中, 它有哪一些恶意属性
